# Стенд: демонстрация SQL-инъекции

Этот проект сделан для практической работы в университете.

Важно:
- Это статический демонстрационный сайт.
- Здесь нет базы данных и серверной авторизации.

## Что внутри

- Страница логина
- Кнопка входа
- Обычный вход: логин `admin`, пароль `admin`
- Специальный логин для входа: `admin' OR 1 = 1`
- Страница «банковского счета» с балансом 13 000 BTC

## Запуск локально

Просто открой `index.html` в браузере.

## Публикация в GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузите файлы проекта в репозиторий (ветка `main`).
3. Откройте `Settings` -> `Pages`.
4. В `Build and deployment` выберите:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main` и папка `/ (root)`
5. Нажмите `Save`.
6. Через 1-2 минуты откройте ссылку формата:
   `https://<username>.github.io/<repo-name>/`

## Для защиты в реальных проектах

- Использовать параметризованные запросы (prepared statements)
- Никогда не собирать SQL через конкатенацию строк
- Делать проверку авторизации только на сервере
- Не хранить пароли в открытом виде
